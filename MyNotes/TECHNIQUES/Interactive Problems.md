# 📝 General Guide: Solving Interactive Problems

ইন্টারঅ্যাক্টিভ প্রবলেম সাধারণ প্রবলেমের চেয়ে আলাদা, কারণ এখানে জাজ (Judge) এবং তোমার প্রোগ্রামের মধ্যে রিয়েল-টাইম কথোপকথন হয়। নিচে এর মূল বিষয়গুলো দেওয়া হলো:

## ১. আউটপুট ফ্লাশিং (Essential)

ইন্টারঅ্যাক্টিভ প্রবলেমে আউটপুট দেওয়ার পর বাফার ক্লিয়ার বা ফ্লাশ করা বাধ্যতামূলক।

- **কেন?** ফ্লাশ না করলে তোমার কুয়েরি জাজের কাছে পৌঁছাবে না, ফলে জাজ উত্তর দেবে না এবং তুমি **Idleness Limit Exceeded** খাবে।
    
- **পদ্ধতি:** তোমার টেমপ্লেটে `#define endl '\n'` থাকলে সেটি রিমুভ করো। `std::endl` ব্যবহার করলে তা অটো-ফ্লাশ করে।
    
- **Fast I/O:** `ios_base::sync_with_stdio(0); cin.tie(0);` ইন্টারঅ্যাক্টিভ প্রবলেমে ব্যবহার না করাই ভালো।
    

## ২. কুয়েরি লিমিট (Query Limit)

প্রতিটি প্রবলেমে কুয়েরি করার একটি নির্দিষ্ট সীমা থাকে (যেমন: $N+2$ বা $2 \cdot \log_2 N$)।

- কুয়েরি করার সময় একটি কাউন্টার রাখা ভালো যাতে লিমিট পার না হয়।
    
- মনে রাখবে, **Answer** প্রিন্ট করা সাধারণত কুয়েরি লিমিটের মধ্যে গণ্য হয় না।
    

## ৩. জাজ এরর হ্যান্ডলিং (-1 Check)

যদি তোমার কুয়েরি ভুল হয় বা লিমিট শেষ হয়ে যায়, জাজ সাধারণত `-1` রিটার্ন করে।

- **সতর্কতা:** তোমার কোড যদি `-1` পাওয়ার পর তা প্রসেস করতে থাকে, তবে তা ইনফিনিট লুপে পড়ে যাবে।
    
- **সমাধান:** প্রতিবার ইনপুট নেওয়ার সময় চেক করো: `if (res == -1) exit(0);`
    

## ৪. বাইনারি সার্চ ইন ইন্টারঅ্যাক্টিভ (Binary Search)

অধিকাংশ ইন্টারঅ্যাক্টিভ প্রবলেম বাইনারি সার্চ দিয়ে সমাধান করা যায়।

- **Fixed Range Query:** $[1, mid]$ রেঞ্জে কুয়েরি করা সবসময় নিরাপদ। এতে ভুল হওয়ার সম্ভাবনা কম থাকে।
    
- **Monotonicity:** কুয়েরির রেজাল্ট কি বাড়ছে না কমছে (যেমন: Sum, XOR, Max) তা খেয়াল রেখে `low` এবং `high` আপডেট করো।
    

## ৫. তোমার টেমপ্লেট অনুযায়ী হেল্পার ফাংশন

তোমার `gap` ম্যাক্রো ব্যবহার করে একটি জেনারিক কুয়েরি স্ট্রাকচার:

```cpp
// General Query Template
auto ask = [&](vector<int> args) {
    cout << "?";
    for(auto x : args) cout << " " << x;
    cout << endl; // Auto-flushes
    int res;
    if (!(cin >> res) || res == -1) exit(0);
    return res;
};

// General Answer Template
auto answer = [&](vector<int> ans) {
    cout << "!";
    for(auto x : ans) cout << " " << x;
    cout << endl;
};
```

## ৬. ডিবাগিং টিপস (Local Testing)

ইন্টারঅ্যাক্টিভ প্রবলেম লোকালি ডিবাগ করা কঠিন।

- ম্যানুয়ালি ইনপুট দিয়ে জাজের মতো অভিনয় করে টেস্ট করো।
    
- ভুল আউটপুট দিলে কেন দিচ্ছে তা দেখার জন্য `cerr` (Error Stream) ব্যবহার করতে পারো, কারণ জাজ `cerr` ইগনোর করে কিন্তু তুমি কনসোলে আউটপুট দেখতে পাবে।
