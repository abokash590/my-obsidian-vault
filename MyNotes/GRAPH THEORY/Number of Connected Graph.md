একটি ডিসকানেক্টেড গ্রাফে কতগুলো আলাদা আলাদা অংশ (Components) আছে, তা বের করাই হলো এই টপিকের মূল লক্ষ্য। গ্রাফটি যদি পুরোপুরি কানেক্টেড হয়, তবে কম্পোনেন্ট সংখ্যা হবে **১**।

---

### **🔍 মূল ধারণা (The Logic)**

১. আমরা ১ থেকে $N$ পর্যন্ত প্রতিটি নোড চেক করি।

২. যদি কোনো নোড আগে **Visited** না হয়ে থাকে, তার মানে আমরা একটি নতুন "Component" খুঁজে পেয়েছি।

৩. সাথে সাথে `cnt++` করি এবং ওই নোড থেকে একটি **BFS** শুরু করি।

৪. এই BFS-টি ওই নির্দিষ্ট কম্পোনেন্টের সব নোডকে `visited` মার্ক করে দেয়।

৫. পরবর্তী লুপে যখন আমরা ওই কম্পোনেন্টের অন্য নোডগুলো পাই, সেগুলো অলরেডি ভিজিটেড থাকায় নতুন করে আর BFS শুরু হয় না।

### **🚀 Optimized Function Template**

আপনার কোডটিকে আরও কমপ্যাক্ট এবং দ্রুত ব্যবহারের উপযোগী করে নিচে দেওয়া হলো:

```cpp
int count_connected_components(int n, vector<int> adj[]) {
    vector<int> vis(n + 1, 0);
    int cnt = 0;

    for (int i = 1; i <= n; i++) {
        if (!vis[i]) {
            cnt++; // নতুন একটি কম্পোনেন্ট পাওয়া গেছে
            
            // BFS part to mark all nodes in this component
            queue<int> q;
            q.push(i);
            vis[i] = 1;

            while (!q.empty()) {
                int u = q.front();
                q.pop();

                for (int v : adj[u]) {
                    if (!vis[v]) {
                        vis[v] = 1;
                        q.push(v);
                    }
                }
            }
        }
    }
    return cnt;
}
```

---

### **⏱️ Complexity Analysis**

- **Time Complexity:** $O(V + E)$
    - যদিও বাইরে একটি লুপ আছে, কিন্তু প্রতিটি নোড এবং প্রতিটি এডজ পুরো প্রসেসে ঠিক একবারই প্রসেস হয়। তাই এটি গ্রাফের সাইজের সমান সময়ে কাজ করে।
- **Space Complexity:** $O(V)$
    - `vis` অ্যারে এবং কিউ (Queue)-এর জন্য।

### **💡 Pro-Tips for Your Note**

- **Component Storage:** আপনি যদি জানতে চান কোন নোড কোন কম্পোনেন্টে আছে, তবে একটি `component_id[]` অ্যারে রাখতে পারেন।
- **Graph is a Tree?** আমরা আগে শিখেছিলাম ট্রি হওয়ার শর্ত। এখন আপনি বলতে পারেন:
    একটি গ্রাফ **Tree** হবে যদি:
    ১. `count_connected_components == 1` (কানেক্টেড)
	২. `edges == n - 1` (সাইকেল নেই)
- **Forest:** যদি একটি গ্রাফে অনেকগুলো কম্পোনেন্ট থাকে এবং প্রতিটিতে কোনো সাইকেল না থাকে, তবে সেই গ্রাফকে **Forest** বলা হয়।
