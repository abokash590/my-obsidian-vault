**Related Topics:** `[[DFS]]`, `[[Tree Traversal]]`, `[[Tree DP]]`

একটি ট্রিতে কোনো নির্দিষ্ট নোড $u$ এবং তার নিচের সব চাইল্ড (বংশধর) মিলে মোট কয়টি নোড আছে, তা বের করাই হলো **Subtree Size** ক্যালকুলেশন।

---

### 🚀 Optimized Subtree Size Code


```cpp
#define N  100005
vector<int> adj[N],sz(N);
void dfs(int u,int par) {
    sz[u]=1;
    for (int v:adj[u]) {
        if(v!=par){
            dfs(v,u);
            sz[u]+=sz[v];
        }
    }
}
```

---

### 🔍 লজিক বিশ্লেষণ (How it works)

১. **Bottom-up Approach:** এটি মূলত লিফ নোড (Leaf Node) থেকে কাজ শুরু করে। লিফ নোডের কোনো চাইল্ড থাকে না, তাই তাদের `sz[u] = 1` থাকে।

২. **Recursive Summation:** যখন কোনো নোড তার চাইল্ডের `dfs` শেষ করে ফিরে আসে, সে চাইল্ডের সাবট্রিতে যতগুলো নোড ছিল (`sz[v]`), তা নিজের ভ্যালুর সাথে যোগ করে দেয়।

৩. **Parent Check:** `v != par` নিশ্চিত করে যে আমরা ট্রাভার্সাল করার সময় উল্টো পথে (প্যারেন্টের দিকে) ফিরে যাচ্ছি না।

---

### ⏱️ Complexity

- **Time Complexity:** $O(V + E)$ — ট্রির ক্ষেত্রে এটি মূলত $O(N)$।
- **Space Complexity:** $O(N)$ — `adj`, `sz` এবং রিকার্সন স্ট্যাকের জন্য।

### 💡 এটি কোথায় কাজে লাগে?

- **Problem Solving:** যেমন, "একটি নির্দিষ্ট এডজ (Edge) মুছে ফেললে গ্রাফটি কত বড় দুটি ভাগে ভাগ হবে?" - এটি বের করতে একটি সাবট্রির সাইজ জানলেই চলে।
