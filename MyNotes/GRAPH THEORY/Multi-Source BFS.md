

**Related Topics:** `[[BFS]]`, `[[Shortest Path]]`, `[[Grid Traversal]]`

সাধারণ BFS-এ আমরা একটি মাত্র সোর্স থেকে যাত্রা শুরু করি। কিন্তু **Multi-Source BFS**-এ আমরা একসাথে একাধিক নোডকে সোর্স হিসেবে ধরে কাজ শুরু করি। এটি মূলত সেই সব ক্ষেত্রে ব্যবহার করা হয় যেখানে জানতে চাওয়া হয়: _"যেকোনো একটি সোর্স থেকে নিকটতম দূরত্ব কত?"_

---

### 🔍 মূল ধারণা (The Core Concept)

১. **সব সোর্স একসাথে কিউতে:** লুপের শুরুতে সব সোর্স নোডকে `queue`-তে পুশ করতে হয় এবং তাদের `lvl` বা দূরত্ব `0` সেট করতে হয়।

২. **ভিজিটেড মার্ক:** সোর্সগুলোকে শুরুতেই `vis[v] = 1` করতে হয় যাতে তারা একে অপরকে ভিজিট না করে।

৩. **একইসাথে বিস্তার:** পানির ঢেউয়ের মতো সব সোর্স থেকে একসাথে ট্রাভার্সাল শুরু হয়। ফলে যেকোনো নোডে প্রথম যে সোর্স থেকে ঢেউ পৌঁছাবে, সেটিই হবে ওই নোডের জন্য **সবচেয়ে কাছের সোর্সের দূরত্ব**।

### 🚀 Optimized Function Template

আপনার কোডটিকে আপনার সিগনেচার স্টাইলে (কম্প্যাক্ট) নিচে দেওয়া হলো:

```cpp
vector<int> multiple_src(vector<int>src, vector<int> adj[], int n){
    vector<int>lvl(n+1,-1),vis(n+1,0);
    queue<int>q;
    for(auto v:src){
        q.push(v);
        vis[v]=1;
        lvl[v]=0;
    }
    while(!q.empty()){
        int u=q.front();q.pop();
        for(auto v:adj[u]){
            if(!vis[v]){
                vis[v]=1;
                lvl[v]=lvl[u]+1;
                q.push(v);
            }
        }
    }return lvl;
}
```

---

### ⏱️ Complexity Analysis

- **Time Complexity:** $O(V + E)$
    
    - সোর্স সংখ্যা যতগুলোই হোক না কেন, প্রতিটি নোড এবং এডজ ঠিক একবারই প্রসেস হয়। তাই এটি সাধারণ BFS-এর মতোই ফাস্ট।
        
- **Space Complexity:** $O(V)$

### 💡 বাস্তব জীবনের উদাহরণ (Real-world Analogy)

মনে করুন একটি শহরে অনেকগুলো ফায়ার সার্ভিস স্টেশন আছে। শহরের কোনো এক জায়গায় আগুন লাগলে, সব স্টেশন থেকে গাড়ি বের হবে। আগুন লাগা জায়গায় সবার আগে কোন গাড়ি পৌঁছাবে? অবশ্যই যে স্টেশনটি সবচেয়ে কাছে। **Multi-Source BFS** ঠিক এই শর্টেস্ট ডিস্টেন্সটাই ক্যালকুলেট করে।

### 📌 ব্যবহারের ক্ষেত্র (Use Cases)

- **Nearest Hospital/ATM:** একটি ম্যাপে অনেকগুলো হাসপাতালের লোকেশন দেওয়া আছে, আপনার কারেন্ট লোকেশন থেকে সবচেয়ে কাছেরটির দূরত্ব কত?
    
- **Burning Forest Problem:** যদি বনের কয়েক জায়গায় একসাথে আগুন লাগে, তবে পুরো বন পুড়তে কত সময় লাগবে?
    
- **Multi-Source Grid BFS:** গ্রিডে অনেকগুলো সোর্স থাকলে `dx`, `dy` ব্যবহার করে সহজেই সমাধান করা যায়।
